# Рабочие промты для Copilot

> Короткие команды для типовых задач с прототипами.
> Копируй нужный промт → вставляй в чат → заменяй `<плейсхолдеры>`.

---

## 1. Применить патч-файл в DEV

Используй когда есть готовый файл патча в `materials/` и нужно применить его к Angular-прототипу.

```
Применить патч к прототипу.

Файл патча: materials/<путь к файлу патча>.md
Целевой прототип: src/app/prototypes/<slug>/

Инструкция:
1. Прочитай файл патча целиком
2. Изучи текущий код целевого прототипа
3. Адаптируй изменения из патча (он может быть в React/Next.js) под Angular 16 + Tailwind
4. Примени все изменения из патча последовательно
5. Обнови changelog: добавь запись в `src/app/prototypes/<slug>/changelog.data.ts`
6. Убедись что `npm run build` проходит без ошибок
7. Закоммить в ветку dev: git add . && git commit -m "feat(<slug>): <краткое описание патча>"

Используй мультиагентную систему и to-do для отслеживания.
```

### Примеры использования

**PUDU плагин, патч v1.6:**
```
Применить патч к прототипу.

Файл патча: materials/Pudu роботы/Промты/Плагин/Промт_прототипа_PUDU_плагин_iikoFront_v1.6_патч.md
Целевой прототип: src/app/prototypes/front-pudu-plugin/
```

**PUDU админка, патч v1.8:**
```
Применить патч к прототипу.

Файл патча: materials/Pudu роботы/Промты/Админ Панель/Промт_прототипа_PUDU_Admin_панели_v1.8_патч.md
Целевой прототип: src/app/prototypes/pudu-admin/
```

## 2. Применить корректировки из чата в DEV

Используй когда описываешь изменения прямо в чате (без файла патча).

```
Применить корректировки к прототипу <slug>.

Изменения:
- <описание изменения 1>
- <описание изменения 2>
- ...

Инструкция:
1. Изучи текущий код прототипа src/app/prototypes/<slug>/
2. Примени все перечисленные изменения
3. Обнови changelog: добавь запись в `src/app/prototypes/<slug>/changelog.data.ts`
4. Убедись что `npm run build` проходит без ошибок
5. Закоммить в ветку dev: git add . && git commit -m "fix(<slug>): <краткое описание>"

Используй мультиагентную систему и to-do для отслеживания.
```

### Пример использования

```
Применить корректировки к прототипу front-pudu-plugin.

Изменения:
- На экране списка заданий добавить колонку "Приоритет"
- В форме создания задания сделать поле "Робот" обязательным
- Исправить отображение статуса — использовать UiBadge вместо текста
```

## 3. Опубликовать DEV в MASTER (деплой)

Используй когда все изменения в DEV проверены локально и готовы к деплою на view21.ru.
Промт полностью автоматический — никакого ручного участия не требуется.

```
Опубликовать изменения из DEV в MASTER.

Инструкция (выполни ВСЁ автоматически, без вопросов и пауз):

1. Убедись что текущая ветка — dev: `git checkout dev`
2. Убедись что нет незакоммиченных изменений: `git status`
   — Если есть — закоммить: `git add . && git commit -m "wip: незакоммиченные изменения"`
3. Выполни `npm run build` — если сборка падает, ОСТАНОВИ процесс и сообщи об ошибке
4. В каждом `src/app/prototypes/<slug>/changelog.data.ts` где есть `status: 'unreleased'`:
   — Смени `status: 'unreleased'` → `'released'`
   — Установи `date` на сегодняшнюю дату (формат YYYY-MM-DD)
5. Закоммить changelog-изменения: `git add . && git commit -m "docs: changelogs для релиза YYYY-MM-DD"`
6. Запушь dev: `git push origin dev`
7. Переключись на master: `git checkout master`
8. Подтяни актуальный master: `git pull origin master`
9. Вмерживай dev в master: `git merge dev`
   — Если конфликты — разреши их автоматически (приоритет dev), закоммить
10. Запуш master: `git push origin master`
11. Вернись на dev: `git checkout dev`
12. Выведи финальную сводку:
    — Какие прототипы затронуты
    — Сколько коммитов влито в master
    — Краткое описание изменений
    — Подтверждение: «master обновлён, деплой на view21.ru запущен автоматически»
```

## Шпаргалка

| Ситуация | Промт |
|----------|-------|
| Есть файл патча в `materials/` | **#1 — Применить патч** |
| Описываю правки в чате текстом | **#2 — Корректировки из чата** |
| Всё готово, пора деплоить | **#3 — DEV → MASTER (автоматический)** |
