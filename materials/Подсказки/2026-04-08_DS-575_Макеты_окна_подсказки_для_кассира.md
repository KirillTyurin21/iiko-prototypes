## КОНТЕКСТ ПРОЕКТА

### Что такое "Подсказки" (Hints)

Подсказки — функционал Digital Signage для iiko Front (POS-система для ресторанов). Администратор в веб-конструкторе (iikoWeb) создаёт «карточки подсказок»:
- Привязывает **блюда-триггеры** (группы / категории / конкретные блюда) — левая часть
- Привязывает **одно блюдо-рекомендацию** — правая часть (ограничение v1: «Многие к одному»)
- Задаёт **слоган** (до 250 символов), **изображение** (из внешнего меню / галереи / загрузка), **скидку** (Flexible-скидку iiko)
- Назначает подсказку на **терминалы** (поштучно, галочками)

Когда кассир добавляет блюдо-триггер в заказ, срабатывает триггер:
- **На Customer Screen** — появляется контрол подсказки в настроенной Области (Area)
- **На экране iikoFront (кассира)** — появляется модальное диалоговое окно с предложением добавить рекомендуемое блюдо

### Архитектура: Area + Control (из продукта Rivals)

> «Подсказки реализуются как **элемент Темы** Customer Screen (аналогично существующему элементу «Анимации») и используют архитектуру отображения **Область (Area) + Контролы (Control)**, портированную из продукта Rivals.»

- **Область (Area)** — контейнер для контролов подсказок на экране покупателя
- **Контрол (Control)** — визуальный шаблон подсказки (тип `'HINT'`, неизменяемый после создания)
- **Триггер** — событие добавления блюда в чек
- **Плагин CS** — обработчик триггеров на стороне iikoFront

### Двойное отображение (ключевое решение от 09.02.2026)

> «Мы должны подсказывать подсказку **двум людям**. Во фронте должно появляться диалоговое окно с подсказкой. И на экране у клиента в кастомер скрина появляется подсказка. Там, когда ты на фронте, у тебя появляется вот этот "чувак, предложи пирожок" и кнопка — добавить, не добавить. Как только ты там нажал добавить, тебе уже не надо искать никакие пирожки в меню.» — **Руслан**

### Модель данных подсказки (Server → Plugin payload)

Плагин получает от сервера JSON со следующей структурой:

```json
{
  "hints": [
    {
      "id": "hint-uuid",
      "triggers": [
        { "type": "group|category|dish", "id": "product-id" }
      ],
      "recommendation": {
        "id": "recommended-product-id",
        "name": "Пирожок вишнёвый"
      },
      "imageUrl": "https://...",         // Может быть null
      "slogan": "Не забудьте купить пирожок!\nПри покупке с кофе — скидка 15%",
      "discountId": "discount-uuid",     // Flexible-скидка iiko (опционально)
      "discount_type": "percent|fixed_sum",  // v2 — сохранено в модели
      "discount_value": 15.0,               // v2 — сохранено в модели
      "period": { "start": "2026-01-01T00:00:00Z", "end": "2026-12-31T23:59:59Z" },
      "time":   { "start": "08:00", "end": "23:00" }
    }
  ]
}
```

**Механизм imageUrl:**
- Если `imageSource = "gallery"` или `"uploaded"` → URL готов
- Если `imageSource = "menu"` → сервер резолвит URL из внешнего меню по `recommendation.id`
- Если `imageSource = null` → `imageUrl = null`, плагин показывает заглушку
- Fallback: если `imageUrl == null && recommendation != null` → `GET /api/v1/products/{recommendation.id}/image`

### Элементы контрола подсказки (16 типов в конструкторе Тем CS)

Для Customer Screen доступны следующие элементы контрола (тип `'HINT'`):

**Переиспользуемые (10):** `area`, `text`, `image`, `productImage`, `productName`, `productFullName`, `productForeignName`, `price`, `productDescription`, `productForeignDescription`

**Уникальные для Подсказок (6):**
| Элемент | Описание | Атрибуты |
|---------|---------|----------|
| `hintSlogan` | Слоган подсказки (до 250 символов) | Layout, Border, Font |
| `hintBanner` | Баннер подсказки (PNG/JPG). **Обязателен** — добавляется автоматически, не удаляется (БП-7) | Layout, Border |
| `discountName` | Название скидки из справочника iiko | Layout, Border, Font |
| `discountAmount` | Размер скидки (сумма с валютой) | Layout, Border, Font |
| `oldPrice` | Цена до скидки (перечёркнутая) | Layout, Border, Font |
| `discountedPrice` | Цена со скидкой (вычисляемое: Цена − Скидка) | Layout, Border, Font |

> **Важно для макетов:** Эти элементы доступны в конструкторе для Customer Screen. Окно для **кассира** (iikoFront) — отдельная реализация, но должно визуально отображать те же данные: слоган, картинку, цену, скидку, название блюда.

### Механизм скидок (Flexible-скидка iiko)

> «Подсказка ради подсказки не имеет никакого смысла — она сильно не сработает, если ты не дашь человеку скидку.»

- API iikoFront (SDK) **не возвращает размер/процент скидки** через `GetDiscountByID`
- Решение: пользователь создаёт **одну Flexible-скидку** (назначаемую, без фиксированного %) в бэк-офисе iiko
- Плагин при синке получает: `discount_id`, `discount_type`, `discount_value`
- В v1: только стандартная фиксированная Flexible-скидка, без выбора типа

## ОБЯЗАТЕЛЬНЫЕ ТРЕБОВАНИЯ К МАКЕТАМ

### Структура модального окна

| Элемент | Обязательность | Описание |
|---------|---------------|----------|
| **Заголовок (title)** | Обязательно | Название подсказки. Пример: "Специальное предложение" |
| **Слоган (slogan)** | Обязательно | Короткий рекламный текст (до 250 символов). Пример: "Купи яблоко на 7 рублей дешевле!" |
| **Описание (description)** | Опционально | Расширенный текст. Может быть пустым. По умолчанию: что вызвало триггер + что предлагается |
| **Картинка блюда (imageUrl)** | Опционально | Изображение рекомендуемого блюда. Некрупное, адекватное интерфейсу. Источник: внешнее меню / галерея / загрузка. Может отсутствовать (заглушка) |
| **Атрибуты** | Опционально | Аллергены, состав, калорийность — если есть в данных. Можно вынести мелким шрифтом |
| **Цена (price)** | Опционально | Текущая цена рекомендуемого блюда |
| **Старая цена (oldPrice)** | Опционально | Цена до скидки (перечёркнутая). Показывается при наличии скидки |
| **Цена со скидкой (discountedPrice)** | Опционально | Вычисляемое: Цена − Скидка. Показывается при наличии скидки |
| **Название скидки (discountName)** | Опционально | Название скидки из справочника iiko. Пример: "Скидка на яблоко" |
| **Размер скидки (discountAmount)** | Опционально | Сумма скидки с валютой. Пример: "–7 руб." |
| **Кнопка "Добавить"** | Обязательно | Добавляет блюдо-предложение в заказ. Крупная, заметная. **Должна содержать название блюда и цену** (если доступна). Пример: "Добавить Яблоко Голден — 49 руб." |
| **Кнопка "Отказаться"** | Обязательно | Закрывает окно без действия. Тоже крупная (кассир должен быстро отказать, если клиент не хочет) |

### Поведение

| Параметр | Значение |
|----------|----------|
| **Тип окна** | Модальное (блокирующее весь интерфейс) |
| **Расположение** | По центру экрана кассы |
| **Размер** | Крупное - перекрывает область выбора меню |
| **Закрытие** | ТОЛЬКО по кнопкам. Клик вне окна НЕ закрывает его |
| **Затемнение фона** | Да, полупрозрачный оверлей за окном |
| **Задержка** | Появляется через 5-10 секунд после добавления триггер-блюда |
| **Очередь** | Если несколько подсказок — выводятся по очереди (FIFO). Подсказки **накапливаются**, НЕ пропадают при переключении на другое блюдо |
| **Удаление триггера** | При удалении блюда-триггера из чека — соответствующая подсказка скрывается |
| **Переполнение** | Новая подсказка вытесняет **самую старую** (FIFO) — актуально для Customer Screen, на кассире окна идут по очереди |

> **Обоснование FIFO (Михаил, 06.03.2026):** «Подсказка моргнёт, человек не прощает»

### Контекст устройства

- **Экран:** Сенсорный монитор iiko Front (кассовый терминал)
- **Разрешение:** Типичное 1920x1080 или 1280x1024 (кассовые мониторы)
- **Ориентация:** Горизонтальная (landscape)
- **Ввод:** Сенсорный (пальцами), кнопки должны быть достаточно крупными для тач-интерфейса
- **Тёмная тема:** Интерфейс iiko Front чаще всего в тёмных тонах (тёмно-серый, чёрный фон). Макет должен контрастировать с тёмным фоном

### Визуальный стиль

- **Тени:** Использовать `box-shadow` для отделения окна от фона (обсуждалось 07.04)
- **Скругления:** `border-radius` для карточки и картинки (обсуждалось 07.04)
- **Полупрозрачность:** Оверлей фона (backdrop) полупрозрачный
- **Цветовая гамма:** Нейтральная/корпоративная. Кнопка "Добавить" - яркий акцентный цвет (зелёная/синяя). Кнопка "Отказаться" - нейтральная (серая/белая рамка)
- **Шрифты:** Чистые, читаемые sans-serif. Заголовок крупный, описание среднее

## НАПРАВЛЕНИЯ ДИЗАЙНА (рекомендации)

### Вариант 1: Минимализм
- Фокус на тексте, минимум графики
- Крупный слоган, чёткие кнопки
- Картинка маленькая или отсутствует
- Быстрое восприятие (кассир читает за 2 секунды)
- **Для кого:** Заведения с высоким потоком, где скорость критична

### Вариант 2: Визуальный / продающий
- Крупная аппетитная картинка блюда
- Цена перечёркнутая -> новая цена (как в интернет-магазинах)
- Яркий акцент на кнопке "Добавить"
- Больше места под визуал, меньше текста
- **Для кого:** Заведения с фокусом на допродажи, где визуал решает

### Вариант 3: Сбалансированный / карточный
- Карточка товара: картинка слева, текст справа (или наоборот)
- Чёткое разделение зон (заголовок / контент / действия)
- Атрибуты и описание видны, но не доминируют
- **Для кого:** Универсальный вариант, баланс информации и читаемости

## ВАЖНЫЕ ЗАМЕЧАНИЯ

1. **НЕ делать анимацию** — анимацией занимается Алиса отдельно, в макетах только статичное состояние
2. **НЕ проектировать админку** — только окно на стороне кассира (iiko Front)
3. **Учитывать пустые поля** — описание и картинка могут отсутствовать. Макет должен выглядеть хорошо и без них
4. **Кнопки для пальцев** — минимальный тач-таргет ~48px, но лучше больше (это кассовый сенсорный монитор)
5. **Тёмный фон** — интерфейс iiko Front тёмный. Модальное окно контрастно светлое или с полупрозрачным тёмным фоном
6. **Масштаб** — окно должно занимать существенную часть экрана (60-80% ширины), но НЕ весь экран
7. **Очередь подсказок** — если у блюда несколько триггеров, окна показываются по очереди. Макет одного окна, не стопки
