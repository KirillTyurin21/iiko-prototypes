## Детальное описание доработок

### D-01 [P0] Disabled-состояние подключённых касс

**Что сейчас:** Все терминалы в чекбокс-списке одинаково кликабельны. Нет понимания, какие кассы уже подключены в другой заявке.

**Что нужно:**
- Добавить поле `isConnected: boolean` и `connectedDate?: string` в интерфейс `AvailableTerminal`
- Кассы с `isConnected: true` — чекбокс `disabled`, визуально приглушены
- Рядом с названием пометка: «(Подключена DD.MM.YYYY)»
- Disabled-кассы не участвуют в счётчике доступных

**Затрагивает:**
- `types.ts` — расширить `AvailableTerminal`
- `data/mock-data.ts` — обновить `MOCK_STORE_TERMINALS`
- `comet-main-screen.component.ts` — шаблон чекбоксов + логика toggle

---

### D-02 [P0] Лимит 10 касс на одну заявку (Merchant ID)

**Что сейчас:** Можно выбрать любое количество касс — ограничений нет.

**Что нужно:**
- Константа `MAX_TERMINALS_PER_MERCHANT = 10`
- После выбора 10 касс — оставшиеся невыбранные чекбоксы становятся `disabled` с пометкой «(лимит 10)»
- Счётчик: «Выбрано: X из 10 (максимум)» или «Выбрано: X из M доступных»
- Валидация: минимум 1 касса, максимум 10

**Затрагивает:**
- `comet-main-screen.component.ts` — шаблон, `toggleEntryTerminal()`, счётчик

---

### D-03 [P0] Вкладка «Подключение» видна без OAuth

**Что сейчас:** Все три вкладки (Организации, Заявки, Подключение) обёрнуты в `*ngIf="oauthState.isAuthorized"`. Без авторизации OAuth-секция вообще не видна.

**Что нужно по решению встречи (P-07):** Вкладка «Подключение» (текущие рестораны с настройками) должна быть доступна **всегда**, даже если пользователь не авторизован через OAuth. Заказчик сказал: «Не закрывать раздел с подключениями».

**Варианты реализации:**
- Вынести вкладку «Подключение» из блока `*ngIf="oauthState.isAuthorized"` наружу
- Или показывать табы всегда, но на вкладках «Организации» и «Заявки» — плашка «Авторизуйтесь», а «Подключение» — доступна

**Затрагивает:**
- `comet-main-screen.component.ts` — структура шаблона OAuth-секции

---

### D-04 [P1] Сценарий >10 касс — повторная заявка на тот же RMS

**Что сейчас:** Один и тот же ресторан (storeId) нельзя выбрать дважды в одной форме — `getAvailableStores()` исключает уже использованные. Но между заявками (когда предыдущая уже отправлена) можно выбрать повторно. Однако без disabled-касс (D-01) нет понимания, какие уже заняты.

**Что нужно:**
- При выборе ресторана, у которого уже есть активная заявка — подключённые кассы отображаются `disabled` (см. D-01)
- Пользователь подаёт новую заявку только на свободные кассы
- Ограничение 10 касс (D-02) работает в рамках **новой** заявки

**Зависит от:** D-01 (disabled-кассы), D-02 (лимит 10)

**Затрагивает:**
- `comet-main-screen.component.ts` — логика подгрузки касс с учётом уже отправленных заявок

### D-05 [P1] Аккордеон — сворачивание блоков точек

**Что сейчас:** Все карточки точек всегда развёрнуты. При 3+ точках форма очень длинная.

**Что нужно:**
- Добавить поле `expanded: boolean` в `MerchantEntry` (или управлять локальным состоянием)
- Заголовок каждого блока «Торговая точка N» — кликабельный, сворачивает/разворачивает содержимое
- При нажатии «Добавить новую точку» — предыдущие блоки сворачиваются, новый — развёрнут
- В свёрнутом состоянии показывать краткую сводку: название точки, кол-во выбранных касс
- Кнопка «Удалить» в заголовке блока — `stopPropagation`, чтобы не триггерить сворачивание

**Затрагивает:**
- `comet-main-screen.component.ts` — шаблон карточки, `addMerchantEntry()`, новый метод toggle

### D-06 [P2] Переименование кнопки добавления

**Что сейчас:** Текст кнопки: «Добавить ещё торговую точку»

**Что нужно по встрече (N-06):** Заказчик предложил «Добавить новую точку». Также предложил расположить кнопку рядом с «Подать заявку», а не отдельно.

- Переименовать в «Добавить новую точку»
- Переместить из отдельного блока (dashed border) в footer формы рядом с «Подать заявку» и «Отмена»

**Затрагивает:**
- `comet-main-screen.component.ts` — шаблон (текст кнопки, расположение)

### D-07 [P2] Автозаполнение адреса из RMS

**Что сейчас:** При выборе торговой точки автоматически заполняется только `name` (из `storeName`). Поле `address` остаётся пустым.

**Что нужно:**
- Добавить поле `address` в интерфейс `StoreTerminals`
- В `MOCK_STORE_TERMINALS` добавить адреса для каждого ресторана
- В `onStoreSelected()` — автозаполнять `entry.address` из адреса RMS
- Примечание (N-09): «У кассы нет атрибута адреса» — адрес берётся с уровня RMS

**Затрагивает:**
- `types.ts` — расширить `StoreTerminals`
- `data/mock-data.ts` — добавить адреса
- `comet-main-screen.component.ts` — `onStoreSelected()`

### D-08 [P0] Обновление мок-данных

**Что сейчас:** 8 ресторанов по 1–3 кассы. У касс нет `isConnected`. Недостаточно для демонстрации сценариев N-03, N-04, N-07.

**Что нужно — минимум 3 ресторана-кейса:**

| Ресторан | Касс | isConnected | Кейс |
|----------|------|-------------|------|
| Ресторан «Премьер» | 5 | 0 из 5 | Базовый — все кассы свободны |
| Кафе «Уют» | 6 | 2 из 6 | Частично подключены — 2 disabled |
| Кофе-поинт «ЦУМ» | 14 | 10 из 14 | Достигнут лимит — 10 disabled, свободны 4 |

- Расширить `AvailableTerminal`: добавить `isConnected`, `connectedDate`
- Добавить / обновить данные `MOCK_STORE_TERMINALS`
- Можно оставить парт старых ресторанов без изменений (isConnected = false по умолчанию)

**Затрагивает:**
- `types.ts`, `data/mock-data.ts`
