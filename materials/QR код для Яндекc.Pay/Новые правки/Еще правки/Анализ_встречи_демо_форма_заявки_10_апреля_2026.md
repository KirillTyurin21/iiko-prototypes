### Этап 1: Модели данных

**Шаг 1.** [P0] types.ts - новый интерфейс RestaurantBlock

В файле `types.ts` создать интерфейс `RestaurantBlock`, описывающий один блок формы заявки. Блок содержит:
- Привязку к ресторану: `storeId`, `storeName`, `address` (из RMS)
- MCC-код (slug из справочника `MccCode`)
- Банковские реквизиты: БИК, расчетный счет, корреспондентский счет
- Контактные данные: email, телефон, фамилия, имя, отчество
- Флаг `expanded` (boolean) - управление аккордеоном

В существующий интерфейс `PosInfo` (или аналог для кассы) добавить:
- `isConnected: boolean` - true если касса уже привязана к другому мерчанту
- `connectedMerchantDate?: string` - дата привязки (отображается как подсказка)

Добавить константу `MAX_TERMINALS_PER_MERCHANT = 10`.

### Этап 3: Мульти-ресторан

**Шаг 5.** [P1] Структура формы как массив блоков (аккордеон)

Переделать форму заявки так, чтобы она поддерживала несколько ресторанов. Вместо одного набора полей - массив `restaurantBlocks: RestaurantBlock[]`. Каждый блок отображается как сворачиваемая секция (аккордеон):

- Клик по заголовку сворачивает/разворачивает содержимое
- Внутри: селектор ресторана, поля, кассы (как описано в шагах 1-4)

**Шаг 6.** [P1] Кнопки управления блоками

- **"Добавить новую точку"** - внизу формы, рядом с "Подать заявку" и "Отмена". Создает новый блок в массиве

Итоговый набор кнопок внизу формы:

### Этап 4: Логика компонента

**Шаг 7.** [P1] Предзаполнение нового блока

При нажатии "Добавить новую точку":
1. Свернуть предыдущий блок (expanded = false)
2. Создать новый блок с `expanded: true`
3. Скопировать из предыдущего блока значения: БИК, расчетный счет, корр. счет, email, телефон, фамилия, имя, отчество
4. НЕ копировать: storeId, storeName, address, MCC-код - это уникально для каждой точки

---

**Шаг 8.** [P0] Селектор ресторана и автозаполнение

В начало каждого блока добавить выпадающий список (select/dropdown) для выбора ресторана из `MOCK_STORES`.

При выборе ресторана:
1. Автоматически заполнить `storeName` и `address` из данных выбранного ресторана
2. Загрузить и отобразить список касс этого ресторана (секция чекбоксов из шага 3)
3. Сбросить ранее выбранные кассы если ресторан изменился
4. Обновить заголовок блока аккордеона на название ресторана

---

**Шаг 9.** [P0] Валидация формы - canSubmit()

Кнопка "Подать заявку" активна (`enabled`) только когда ВСЕ блоки валидны. Блок считается валидным если:
- Выбран ресторан (storeId не пустой)
- Выбрана хотя бы 1 касса
- Заполнены обязательные поля: БИК, расчетный счет, корр. счет, email, телефон, фамилия, имя
