## 1. Общее описание

Панель управления интеграцией с Яндекс.Пэй в iikoWeb. Позволяет администратору:
- Просматривать организации и рестораны в виде дерева
- Управлять API-ключами Яндекс.Пэй для каждого ресторана
- Привязывать QR-таблички к терминалам (кассам)
- Назначать табличку по умолчанию для ресторана

Это **отдельная страница** внутри iikoWeb, а не часть другого модуля. Прототип полностью автономен — у него собственный header, sidebar и content area.

## 3. Технологический стек

| Что | Версия/Описание |
|-----|-----------------|
| **Framework** | Angular 16 (standalone components) |
| **Styling** | Tailwind CSS + кастомная тема |
| **Icons** | `lucide-angular` через `IconsModule` |
| **Routing** | Angular Router, lazy loading (`loadComponent`) |
| **State** | Локальное состояние компонента (свойства класса) |
| **Persistence** | `StorageService` → `localStorage` |
| **Forms** | `FormsModule` + `[(ngModel)]` (двусторонняя привязка) |
| **Шрифт** | Roboto (локальные .ttf файлы) |
| **Сборка** | Angular CLI, dev-сервер порт 3000 |

### Angular-паттерны
- `standalone: true` — обязательно для всех компонентов
- `inject()` вместо constructor injection
- `*ngIf`, `*ngFor` (Angular 16, **НЕ** `@if`/`@for` — это Angular 17+)
- Inline templates (`template: \`...\``) — отдельные .html файлы не используются
- Импорты зависимостей напрямую в каждый компонент

### 6.6. Диалог подтверждения (очистка ключа)

Используется `<ui-confirm-dialog>` из UI-библиотеки:
```html
<ui-confirm-dialog
  [open]="showDeleteDialog"
  title="Очистить ключ Яндекс.Пэй?"
  [message]="'... для ресторана «' + storeName + '»? ...'"
  confirmText="Очистить"
  cancelText="Отмена"
  (confirmed)="handleClearKey()"
  (cancelled)="showDeleteDialog = false"
></ui-confirm-dialog>
```

### 6.7. Toast-уведомление

Кастомная реализация (не UI-компонент):
```html
<div *ngIf="toastMessage"
  class="fixed bottom-6 right-6 z-50 bg-white border border-gray-200
         rounded-lg shadow-lg px-4 py-3 max-w-sm animate-slide-up">
  <p class="text-sm font-medium text-gray-900">{{ toastMessage }}</p>
  <p *ngIf="toastDescription" class="text-sm text-gray-500 mt-0.5">{{ toastDescription }}</p>
</div>
```
- Автоскрытие через 3 секунды (`setTimeout`)

## 9. Пользовательские сценарии (User Flows)

### Сценарий 1: Первичная настройка ключа
1. Пользователь выбирает ресторан **без ключа** в дереве
2. Detail Panel: поле ключа пустое, подсказка: *«Ключ ещё не настроен»*
3. Пользователь вводит ключ → кнопка «Сохранить» становится активной
4. Нажимает «Сохранить» → ключ сохранён, появляется секция QR-табличек
5. Статус ресторана в дереве: серый → оранжевый (`partial`)

### Сценарий 2: Смена ключа
1. Выбран ресторан **с ключом**
2. Пользователь изменяет значение ключа → «Сохранить» активна
3. Нажимает «Сохранить» → **все привязки терминалов сбрасываются**
4. Toast: *«Ключ изменён / Все привязки терминалов сброшены. Список табличек обновлён.»*

### Сценарий 3: Очистка ключа
1. Выбран ресторан с ключом → нажать «Очистить ключ»
2. Диалог подтверждения → «Очистить»
3. Ключ обнулён, секция QR-табличек скрывается, статус → none

### Сценарий 4: Привязка QR-таблички к терминалу
1. Выбран ресторан с ключом и терминалами
2. В карточке терминала — select с перечнем табличек
3. Выбор → **мгновенное сохранение** (без кнопки Save), toast подтверждает
4. Если табличка уже на другом терминале — суффикс *(на N терминалах)*
5. Счётчик обновляется

### Сценарий 5: Назначение таблички по умолчанию
1. Select «Табличка по умолчанию» → выбор
2. Значение влияет на `isSaveDisabled` (кнопка «Сохранить»)
3. При нажатии «Сохранить» — default account сохраняется вместе с ключом

### Сценарий 6: Поиск
1. Ввод в поле поиска → debounce 300ms
2. Дерево фильтруется по `storeName`
3. Организации с результатами auto-раскрываются
4. Пустой результат → *«Ничего не найдено»*

private showToast(message: string, description?: string): void {

  }, 3000);
}

## 16. Известные упрощения

- Нет реальных API-вызовов — всё на мок-данных с задержкой 300ms
- Терминалы всех ресторанов (кроме storeId=301) = один и тот же набор [Касса 1, 2, 3]
- Ключ для всех ресторанов = `yk_test_1234567890abcdef`
- Нет валидации формата ключа (любая непустая строка)
- Нет пагинации — все организации грузятся сразу
- Привязка терминалов — мгновенная (без Save), а ключ + default account — через «Сохранить»
- Toast — кастомный `setTimeout`, не глобальный сервис
- Кнопки Save/Clear — кастомные `<button>` с Tailwind, не `<ui-button>`
- Select-поля — нативные `<select>`, не `<ui-select>`
